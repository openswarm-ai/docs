An **agent** in OpenSwarm is a Claude-powered AI that can do anything you can do on your computer — read and write files, run shell commands, search the web, control a browser, send emails, manage your calendar — and handle long-running, multi-step tasks autonomously.

Think of each agent as a teammate you can brief on a task and let loose, while you watch it work in real time.

---

## What makes an agent different from a chatbot

A regular chatbot takes a prompt and returns text. An OpenSwarm agent takes a prompt and **acts on it** — it plans, uses tools, reads results, adapts, and keeps going until the job is done or it needs your input.

Agents can:
- **Run shell commands** to install dependencies, run tests, start servers
- **Read and write code** across your entire project
- **Browse the web** via a real browser — navigating, clicking, filling forms
- **Use MCP integrations** — Gmail, Google Calendar, Drive, Sheets, X/Twitter, Reddit, and any custom MCP server you connect
- **Spawn sub-agents** to parallelize work or delegate specialized tasks
- **Generate interactive apps** (Views) that render live on the dashboard

All of this happens locally on your machine. No cloud relay, no third-party backend.

## Agents in the context of OpenSwarm

OpenSwarm isn't about running one agent — it's about running **many at once**.

Each agent lives as a card on the [dashboard canvas](/dashboard/overview). You can have 10+ agents working simultaneously, each in its own git worktree so they never conflict. The dashboard gives you a bird's-eye view of everything that's happening.

### The swarm model

1. You launch an agent from the toolbar and give it a task
2. That agent can **spawn sub-agents** (via the `Agent` tool) — new cards appear on the canvas automatically
3. Sub-agents can spawn **browser agents** that control browser cards visually
4. Agents can **talk to each other** via `InvokeAgent` — querying another agent's full conversation context
5. All tool approvals from all agents surface in one place — approve individually or batch with **Shift+A** / **Shift+D**

### Git worktree isolation

Every agent operates in its own git worktree and branch. This means five agents can edit five different files (or even the same file) in parallel without merge conflicts. When they're done, you review and merge on your terms.

## Agent lifecycle

| Status | Meaning |
|---|---|
| **Draft** | Created but no message sent yet |
| **Running** | Actively processing — reading files, running commands, calling tools |
| **Waiting for approval** | Paused on a tool call that needs your permission |
| **Completed** | Finished its task |
| **Stopped** | You manually stopped it (can be resumed) |
| **Error** | Something went wrong |

## Modes

Agents run in a **mode** that controls their behavior — which tools are available, what system prompt they use, and how they approach tasks. OpenSwarm ships with built-in modes:

- **Agent** — Full access to all tools. The default general-purpose mode.
- **Ask** — Conversational only, no tool use. For quick questions.
- **Plan** — Thinks through a plan before acting. Read-only tools only.
- **App Builder** — Specialized for generating interactive Views.
- **Skill Builder** — Specialized for creating reusable skills.

You can also create **custom modes** with your own system prompts and tool restrictions. Switch modes at any time from the [chat input](/agent/chat-input).

## Models

Each agent can use a different Claude model:

| Model | Best for |
|---|---|
| **Sonnet 4.6** | Fast, capable, good default for most tasks |
| **Opus 4.6** | Most capable, best for complex reasoning and long-horizon tasks |
| **Haiku 3.5** | Fastest and cheapest, good for simple tasks |

You can switch models mid-conversation — the new model takes effect on the next message.

---

## Tool approvals

When an agent wants to use a tool, it may need your permission depending on how that tool is configured:

- **Allow** — runs automatically, no approval needed
- **Ask** — pauses and shows you exactly what it's about to do (the command, the file, the API call) so you can approve or deny
- **Deny** — blocked entirely

You can approve or deny individually, or use **Shift+A** / **Shift+D** to batch-approve/deny all pending requests across every agent on the dashboard.

## Cost tracking

Every agent session tracks token usage and cost in real time. A context window ring in the chat input shows how much of the 200K context window has been consumed.
