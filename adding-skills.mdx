## Overview

Skills are attached at the **agent level** — each agent has its own set of skills independent of other agents. This lets you give each agent exactly the access it needs, and nothing more.

This page covers how to add, configure, and remove skills from any agent.

---

## Adding a Skill

### From the Agent Settings

1. Go to your agent's page in OpenSwarm
2. Click **Settings** → **Skills**
3. Browse available skills or use the search bar
4. Click **Add** next to the skill you want
5. Complete any required authentication (for built-in skills) or configuration (for MCP/custom skills)
6. Click **Save**

The agent will immediately have access to that skill on its next conversation.

---

## Configuring a Skill

Some skills have configuration options that control how the agent uses them. These vary by skill type:

### Built-in Skills
Most built-in skills work out of the box after connecting your account. Some offer optional scoping:

- **Gmail** — you can restrict to read-only vs. full access
- **Google Drive** — limit to specific folders
- **Browser** — set allowed domains or sandboxing rules

### MCP Skills
When adding an MCP server as a skill, you'll configure:
- **Server URL** — where the MCP server is running
- **Transport type** — `stdio`, `SSE`, or `HTTP`
- **Environment variables** — API keys or secrets the server needs
- **Tool allowlist** — optionally restrict which tools from the server the agent can call

### Custom Skills
Custom skills expose configuration defined by their author. See [Custom Skills](/customization/skills/custom-skills) for details.

---

## Skill Permissions

OpenSwarm lets you scope what each skill can do at a granular level. This is useful when you want an agent to use a service but limit its reach.

**Example:** You might give your support agent access to Gmail but restrict it to a specific label — so it only sees support tickets, never your personal inbox.

To configure permissions:
1. Open the skill in your agent's Skills tab
2. Click the **⚙ Configure** icon
3. Adjust the permission settings available for that skill
4. Save

---

## Removing a Skill

1. Open the agent's **Skills** tab
2. Find the skill you want to remove
3. Click **Remove**
4. Confirm — the agent will lose access immediately

<Warning>
Removing a skill mid-conversation may cause the agent to fail if it tries to use that skill. Make changes when the agent is idle.
</Warning>

---

## Skill Priority & Conflicts

If two skills expose tools with similar names, OpenSwarm surfaces both to the agent and lets it decide which to use based on context. In practice, conflicts are rare because tool names are namespaced by skill.

If you're seeing unexpected behavior, check whether two active skills might be overlapping.

---

## Checking What Skills an Agent Has

At any time, you can see an agent's full skill list:
- Go to the agent's page
- Click the **Skills** tab — all active skills are listed with their connection status

You can also ask the agent directly: *"What tools do you have access to?"* — it will tell you.
