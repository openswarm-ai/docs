The agent chat view is a scrollable message history. Messages are grouped into **user** messages (right-aligned) and **assistant** messages (left-aligned), interspersed with tool call groups.

---

## Message types

### User messages

Your prompts appear as right-aligned bubbles with a subtle background. When you send a message with attachments (images, files, selected elements), those are shown inline above or within the bubble.

### Assistant messages

The agent's responses appear as left-aligned bubbles, rendered as Markdown — supporting code blocks, headings, lists, bold/italic, and inline code.

A **thinking indicator** (three animated dots) shows while the agent is processing between messages.

### Streaming

Assistant responses stream in token-by-token in real time. The latest partial response appears at the bottom of the chat as it arrives.

### Tool calls & results

Tool calls (Bash, Read, Write, MCP tools, etc.) appear as compact grouped bubbles between messages. See [Approvals](/agent/approvals) for how pending tool calls work.

---

## Message action bar

Hovering over any message reveals a row of icon buttons. These are hidden at rest and fade in on hover.

### User message actions

| Button | Action |
|---|---|
| **Copy** | Copies the message text to clipboard. Shows a checkmark for 1.5s to confirm. |
| **Edit** | Opens the message in an inline editor (see below). |
| **Bookmark** | Coming soon. |
| **Branch navigator** | If this message has multiple branch variants, `◀ N/N ▶` arrows appear inline to switch between them. |

### Assistant message actions

| Button | Action |
|---|---|
| **Copy** | Copies the response text to clipboard. |
| **Regenerate** | Re-runs the most recent user message that preceded this response, creating a new branch. |
| **Branch chat** | Forks the conversation at this point into a new session on the dashboard. |

---

## Editing a message

Click the **Edit** icon on a user message to enter edit mode. The message text appears in an inline text field. Make your changes, then:

- **Save** (checkmark / Enter) — saves the edit and re-runs the agent from that point, creating a new [branch](/agent/branching).
- **Cancel** (X) — discards the edit and restores the original message.

<Note>
  Editing a message always creates a new branch. The original message and everything after it is preserved on the previous branch.
</Note>

---

## Regenerating a response

Click the **Regenerate** icon on an assistant message to re-run the user message that immediately preceded it. This is equivalent to editing the user message with identical content — it creates a new branch and re-sends the prompt.

---

## Diff viewer

When an agent uses a **Write** or **Edit** tool to modify a file, the result may be shown as a **diff view** — a side-by-side or inline diff of the old vs. new content. This appears as a special card in the tool call group.

---

## Scroll behavior

The chat auto-scrolls to the bottom as new messages arrive, as long as you're already at (or near) the bottom.

If you scroll up to read history while the agent is still running, auto-scroll pauses. A **scroll-to-bottom button** appears in the bottom-right of the chat to jump back.

---

## Resume bubble

If an agent is stopped mid-run, a **Resume** button appears at the bottom of the chat. Clicking it sends a hidden continuation prompt asking the agent to pick up where it left off.
