---
title: "Element Selection"
description: "Click canvas elements to attach them as context to your next agent message."
---

**Element selection mode** lets you click anything on the dashboard canvas — agent cards, messages, tool calls, view cards, browser cards, or individual DOM elements inside a view — and attach it as structured context to your next prompt.

This is how you say _"do something with this specific thing"_ without having to describe it in text.

---

## Activating selection mode

### From the Chat Input

Click the **cursor/select icon** (AdsClick) in the input toolbar. The canvas enters selection mode — cards and elements highlight on hover to indicate they're selectable.

### Auto-activate on new agent

If **Auto-select mode on new agent** is enabled in Settings, selection mode activates automatically every time you open the new agent input in the toolbar.

---

## What you can select

### Agent cards

Click an agent card to select it. The prompt receives:
- The agent's name and session ID
- A hint: `Use InvokeAgent with session_id "..." to query this agent with full conversation context`

### Messages

Click an individual message bubble inside an expanded agent card. The prompt receives the message role, content, and ID.

### Tool calls / Tool groups

Click a tool call or tool group bubble. The prompt receives the tool name, inputs, and result summary.

### View cards (App cards)

Click a view card on the canvas. The prompt receives the view name, output ID, and description.

### Browser cards

Click a browser card. The prompt receives:
- The card's `browser_id`
- The current tab title and URL
- A hint: `Use BrowserAgent with this browser_id to interact with it`

The browser card also **glows** to confirm it was selected as context.

### DOM elements (inside views)

When a view card is open, you can click individual HTML elements within it. The prompt receives:
- The element's CSS selector path
- Tag name and outer HTML (truncated at 500 chars)
- A screenshot of the element
- Key computed styles

---

## Selected elements in the input

Selected elements appear as **chips** above the chat input editor. Each chip shows the element type and label. Click the `×` on a chip to remove it from the selection.

Element screenshots (for DOM elements and browser cards) are attached as inline images alongside the prompt.

---

## How context is injected into the prompt

When you send a message, the selected elements are appended to your prompt text as a structured block:

```
---
Selected UI Elements:

1. [Agent Card] My Research Agent
   ID: abc123
   (Use InvokeAgent with session_id "abc123" to query this agent...)

2. [Browser Card] Google — search results
   browser_id: browser-xyz
   URL: https://google.com/search?q=...
   (Use BrowserAgent with this browser_id to interact with it...)
```

---

## Deactivating selection mode

- Click the select icon again to toggle off
- Close/dismiss the chat input — selection mode is cleared automatically
- Send the message — the selection is cleared after send

---

## Pasting cards as context

If you've copied cards from the dashboard (`⌘C`), pasting (`⌘V`) into the chat input converts those cards into selection context entries — the same as if you had clicked them in selection mode.
