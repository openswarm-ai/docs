## Overview

Writing a great skill from scratch requires knowing how to structure the `SKILL.md`, write effective trigger descriptions, bundle the right resources, and iterate on instructions until the agent's behavior is dialed in.

The **Skill Builder Agent** handles all of this for you. It's a dedicated AI agent built into OpenSwarm that generates, tests, and refines skills through a conversational workflow — you describe what you want, and it produces a working skill.

---

## How to Use It

### Open the Skill Builder

On the **Skills** page, click the **"Build with AI"** button (or the ✨ floating action button in the bottom-right corner). This opens a resizable chat panel with a dedicated Skill Builder session.

### Describe Your Skill

Tell the agent what you want your skill to do. The more specific you are, the better the result.

**Example prompts:**

> *"I want a skill that helps me write professional cover letters. It should ask about the role and company, then produce a formatted letter with a specific tone."*

> *"Build a skill for generating database migration scripts. It should handle Postgres, understand schema changes, and output idempotent SQL."*

> *"Create a skill that formats code review feedback — it should organize comments by severity and suggest specific improvements."*

### What Gets Generated

The Skill Builder produces a complete `SKILL.md` with:
- **Optimized frontmatter** — name and description tuned for accurate triggering
- **Structured instructions** — step-by-step workflows the agent will follow
- **Examples** — input/output pairs that align behavior
- **Bundled scripts** (when needed) — helper code for deterministic tasks

As the agent works, you'll see a **live preview** of the skill in the main panel — including the rendered Markdown content, name, description, and command.

### Save the Skill

Once you're happy with the result, click **Save Skill** in the Skill Builder header. The skill is immediately saved to `~/.claude/skills/` and available to all your agents.

## The Iteration Loop

The Skill Builder doesn't just generate a first draft and stop. It follows a full **create → test → evaluate → improve** loop:

1. **Capture intent** — The agent interviews you to understand what the skill should do, when it should trigger, and what the output should look like.

2. **Write the skill** — It produces a draft `SKILL.md` with frontmatter, instructions, and examples.

3. **Test with eval cases** — It generates realistic test prompts, runs them against the skill, and produces results for you to review.

4. **Evaluate together** — It shows you the outputs and asks for feedback. Empty feedback = looks good. Specific feedback = it knows what to improve.

5. **Improve and repeat** — Based on your feedback, it refines the skill and retests. This continues until you're satisfied.

---

## Description Optimization

The trigger description is the most important part of a skill — it determines whether the agent activates it. The Skill Builder includes an automated optimization loop:

1. It generates **20 eval queries** — a mix of should-trigger and should-not-trigger prompts
2. You review and adjust the eval set
3. It runs an optimization loop that:
   - Tests the current description against the eval queries
   - Proposes improved descriptions
   - Evaluates each candidate on train and held-out test sets
   - Selects the best-performing description (by test score, to avoid overfitting)
4. It updates the skill with the optimized description

This turns description-writing from guesswork into a measured process.

## Tips for Better Results

**Be specific about context.** Instead of *"a skill for documents"*, say *"a skill for creating SOW documents for consulting engagements, with sections for scope, timeline, deliverables, and pricing."*

**Mention tools the skill should use.** If you know the agent should use a specific library, API, or MCP tool, say so. The Skill Builder will bake that into the instructions.

**Share examples.** If you have examples of good output, paste them in. The agent will use them to calibrate the skill's instructions.

**Ask it to iterate.** The first draft is a starting point. Review the preview, tell it what to change, and let it refine.

**Let it test.** The eval/test loop catches issues you might miss by eyeballing the instructions. Let the process run — the feedback cycle is where skills go from "OK" to "great."

---

## Resetting & Starting Over

If you want to start fresh:
- Click the **↻ Reset** button in the Skill Builder header
- This clears the current session and workspace, and initializes a new one
- Your previously saved skills are not affected
