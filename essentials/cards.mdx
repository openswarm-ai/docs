Agent cards are live chat sessions with an AI agent. Each card corresponds to one conversation.

### Collapsed vs. expanded

By default, agent cards start **collapsed** — showing a compact summary bar with the agent name, status, and the most recent tool call or message. Click the card to **expand/collapse** it.

Expand an agent card to:

- Read the full message history
- Send follow-up messages
- Edit, copy, and branch chats

You can set the  **Default agent spawn state in dashboard** in Settings, this determines if a new agent card is expanded or collapsed.

### Resizing

Drag any edge or corner handle to resize the card. Minimum size is enforced so the card always remains usable.

### Dragging

Grab anywhere in an agent card's header to move the card freely on the canvas.

### Sub-agent cards

When an agent spawns sub-agents, child cards are automatically placed near the parent card and visually linked so you can see what's going on. If you would rather have sub-agents remain hidden by default, you can set this via the **Auto-reveal sub-agents** setting.

### Approval bar

If an agent is waiting for tool-call approval, the card shows an inline **Approval Bar** with Approve / Deny buttons. This is always visible regardless of if an agent card is expanded or collapsed.

## App Cards

App cards embed a live **app** — an interactive, full-stack, applications built by agents — directly on the canvas.

### Opening an app

Open the toolbar → click **Add App** (⌘M) → search and select an output from the picker. The app is placed as a new card on the canvas.

### Interacting with an app

You can interact with an app like any normal website — click buttons, fill forms, scroll, etc.

The header shows:

- The view's name and icon
- A **Refresh** button to reload the iframe
- A **Close** button

### Resizing

Drag any edge or corner handle to resize. Minimum card size: 320 × 200 px.

---

## Browser Cards

Browser cards are full in-app web browsers embedded on the canvas. They support manual browsing and can also be driven autonomously by an agent.

### Open a new browser

Click **Browser** in the toolbar (⌘N). The card opens to your configured homepage.

### Tabs

Browser cards support multiple tabs:

- Click **\+** in the tab bar to open a new tab.
- Click any tab to switch to it.
- Drag tabs to reorder them.
- Click **✕** on a tab to close it.

### Navigation

The card has a standard browser address bar:

- Type a URL or search query and press Enter.
- Bare search terms are sent to Google automatically.
- Back (◀), Forward (▶), and Refresh (↺) buttons work as expected.

### Browser agent overlay

When an agent is actively controlling the card, an **overlay** appears on top of the card showing what the agent is doing in real time. Once the agent finishes, the overlay fades out.

### Resizing

Drag any edge or corner handle to resize.

### Referencing a browser within an agent

When sending a message to an agent, enable "select mode" and select a browser on your canvas. When you send the message, this agent will now be able to both access the context of the browser and functionally control the browser.

---

## Card layering

Cards can overlap. Clicking or interacting with a card automatically brings it to the front.
