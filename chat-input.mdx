The **Chat Input** is the editor at the bottom of every agent card. It's where you type prompts, attach context, and control how the agent behaves.

---

## Basics

- **Enter** — send message
- **Shift+Enter** — new line
- While the agent is busy, new messages are **queued** and dispatched automatically when it finishes

---

## `/` Slash commands

Type `/` at the start of a word to open the command picker:

- **Templates** — insert a reusable prompt template (with optional fill-in fields)
- **Skills** — attach a skill as an inline pill. The skill's content is injected into the prompt on send. You can attach multiple.
- **Modes** — switch the agent's mode without leaving the input

---

## `@` Context commands

Type `@` at the start of a word to open the context picker:

- **`@file`** — browse and attach a local file or folder as context
- **`@web`** — force the agent to use WebSearch/WebFetch for this message
- **MCP integrations** — force specific tool groups (e.g. Google Workspace, X/Twitter, Reddit). These appear as branded pills.

---

## Attachments

### Images
Drag and drop, paste from clipboard, or use the attachment button. Thumbnails appear above the input.

### Files
Drag and drop non-image files — they're uploaded and attached as context paths.

---

## Element selection

Click the **select icon** in the input toolbar to enter **element selection mode**. This lets you click anything on the canvas to attach it as context:

- **Agent cards** — attaches the agent's session ID and a hint to use `InvokeAgent`
- **Browser cards** — attaches the `browser_id` and current URL
- **View cards** — attaches the output name and ID
- **Messages, tool calls** — attaches the content and metadata
- **DOM elements inside views** — attaches the selector, HTML, styles, and a screenshot

Selected elements appear as chips above the input. When you send, they're appended to your prompt as structured metadata.

<Tip>
  If you copy dashboard cards with **⌘C** and paste into the input with **⌘V**, they're converted into selection context automatically.
</Tip>

---

## Mode & model selectors

Two pill buttons at the bottom of the input let you switch the **mode** and **model** for this session. See [Agents overview](/agent/overview) for the available options.

---

## Context window

A small ring next to the selectors shows context usage. Hover for the exact count (e.g. `45.2% · 90.4K / 200K`).
