Actions are the capabilities available to your AI agents. Every tool call an agent makes — reading a file, sending an email, searching the web — is an **action**. The Action Library is where you configure which actions are available, how they're authenticated, and what permissions they require.

---

## The Three Layers

OpenSwarm organizes actions into three layers:

<CardGroup cols={3}>
  <Card title="Built-in Actions" icon="lock">
    The core actions enabling a general agent — file operations, search,  commands, browser automation, and more.
  </Card>

  <Card title="Integrations" icon="plug">
    First-party connectors for services like Google Workspace, Twitter/X, and Reddit. These connect via OAuth or credentials.
  </Card>

  <Card title="MCP Actions" icon="store">
    Any MCP (Model Context Protocol) server — installed from the public registry or added manually. This is the primary extension point.
  </Card>
</CardGroup>

---

## How Actions Flow Into Agent Sessions

When an agent session starts, OpenSwarm assembles the full set of available actions:

1. **Core actions** that are enabled are loaded directly into the session.
2. **All other actions** (like web search, Reddit, Twitter/X, etc.) are loaded on-demand when the agent looks for more specialized tools.

Each action is also subject to its [permission policy](/customization/actions/permissions), which determines whether the agent can execute it freely, must ask the user first, or is blocked entirely.
